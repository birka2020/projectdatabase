 Учёт продаж автомобилей — База данных

Практическая работа по информатике: «Проектирование, нормализация и работа с реляционными базами данных»

Проект представляет собой систему учёта выдачи (покупки) автомобилей, разработанную на SQLite. База данных содержит информацию о покупателях,
автомобилях, моделях/комплектациях и фактах выдачи (продажи) автомобилей.

Технологии
Язык: Python 3
БД: SQLite
Инструменты: SQLite Online, GitHub


Структура базы данных

 Таблица buyers (Покупатели)
| Поле | Тип | Описание |
| id | INTEGER | Уникальный идентификатор покупателя (Primary Key) |
| name | TEXT | Имя покупателя |
| age | INTEGER | Возраст покупателя |


 Таблица cars (Автомобили)
| Поле | Тип | Описание |
| id | INTEGER | Уникальный идентификатор автомобиля (Primary Key) |
| name | TEXT | Название модели автомобиля |
| colors | TEXT | Цвет автомобиля |
| country | TEXT | Страна производитель |
| year | TEXT | Год выпуска |


Таблица model (Модели/комплектации)
| Поле | Тип | Описание |
| id | INTEGER | Уникальный идентификатор модели (Primary Key) |
| name | TEXT | Полное название комплектации |
| money | INTEGER | Стоимость (руб) |
| power | INTEGER | Мощность (л.с.) |


Визуализация данных (SQLite Online)
В папке photo представлены скриншоты каждой таблицы, содержащей по 30 записей, полученные с помощью запроса SELECT * FROM название_таблицы.


Связи между таблицами
- `buyers` (покупатели) и `borrowings` (выдачи) связаны один-ко-многим (один покупатель может совершить несколько покупок).
- `model` (модели) и `borrowings` связаны один-ко-многим (одна модель может быть продана много раз).
- `cars` (автомобили) и `borrowings` связаны один-к-одному (каждый конкретный автомобиль обычно продаётся один раз).
